# Reference 项目页面结构分析

> 本文档详细分析 reference 项目中实现的所有页面,包括页面功能、内容组成和技术特点。

## 项目概述

Reference 项目是一个基于 Next.js 15 App Router 的地质勘察信息聚合平台,采用现代化的技术栈构建。项目包含前台展示、用户中心和管理后台三大模块。

### 技术栈
- **框架**: Next.js 15 (App Router)
- **UI 组件**: Shadcn UI + Radix UI
- **样式**: Tailwind CSS
- **图标**: Lucide React
- **状态管理**: React Hooks
- **数据库**: SQLite (开发环境)

### 1.2 登录页面 (`/login`)

**文件位置**: `src/app/login/page.tsx`

**页面类型**: 客户端组件 (`'use client'`)

**主要功能**:
- 提供多种登录方式
- 统一的用户和管理员登录入口
- 根据角色跳转到不同页面

**登录方式**:
1. **用户名登录** - 用户名 + 密码
2. **邮箱登录** - 邮箱 + 密码
3. **手机号登录** - 手机号 + 验证码(60秒倒计时)

**页面功能**:
- 三种登录方式切换(Tabs组件)
- 密码显示/隐藏切换
- 验证码发送和倒计时
- 表单验证(手机号格式、邮箱格式、用户名格式)
- 第三方登录入口(微信、QQ,功能开发中)
- 忘记密码链接
- 注册链接

**角色跳转逻辑**:
- `admin`/`moderator` → `/admin/dashboard`
- `user`/`paid`/`enterprise` → `/dashboard`

**技术特点**:
- 使用 localStorage 存储 token
- 调用 `/api/auth/login` API
- 统一的登录成功处理函数 `handleLoginSuccess`
- 用户名格式验证 `validateUsernameFormat`

### 1.3 注册页面 (`/register`)

**文件位置**: `src/app/register/page.tsx`

**页面类型**: 客户端组件

**主要功能**:
- 提供两种注册方式
- 收集用户基本信息
- 验证用户输入

**注册方式**:
1. **手机号注册**
   - 手机号 + 验证码
   - 验证码60秒倒计时
   - 自动创建账户

2. **邮箱注册**
   - 姓名(必填)
   - 用户名(可选,3-20位)
   - 邮箱(必填)
   - 密码(必填,至少6位)
   - 确认密码

**页面功能**:
- 两种注册方式切换
- 验证码发送和倒计时
- 表单验证(手机号、邮箱、密码、用户名)
- 用户协议和隐私政策勾选
- 注册成功后跳转登录页

**技术特点**:
- 调用 `/api/auth/register` API
- 调用 `/api/auth/send-sms` 发送验证码
- 用户名验证 `validateUsername`
- 开发环境显示验证码

### 1.4 忘记密码页面 (`/forgot-password`)

**文件位置**: `src/app/forgot-password/page.tsx`

**页面类型**: 客户端组件

**主要功能**:
- 提供密码找回服务
- 两种找回方式
- 两步验证流程

**找回方式**:
1. **手机号找回**
   - 步骤1: 验证身份(手机号 + 验证码)
   - 步骤2: 设置新密码

2. **邮箱找回**
   - 发送重置链接到邮箱
   - 邮箱验证和重置

**页面功能**:
- 两种找回方式切换
- 两步流程管理(step状态)
- 验证码发送和倒计时
- 密码显示/隐藏切换
- 密码强度验证(至少6位)
- 密码一致性验证
- 返回登录页链接

**技术特点**:
- 调用 `/api/auth/send-sms` 发送验证码
- 调用 `/api/auth/verify-reset` 验证身份
- 调用 `/api/auth/send-reset-email` 发送邮件
- 调用 `/api/auth/reset-password` 重置密码
- 步骤状态管理
