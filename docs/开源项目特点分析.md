## 项目一：Next.js Boilerplate

**项目信息**
- **仓库**: https://github.com/ixartz/Next-js-Boilerplate
- **Stars**: 12,539 ⭐
- **作者**: Ixartz (CreativeDesignsGuru)
- **定位**: SaaS 应用开发模板

### 核心技术栈

**基础框架**
- Next.js 16.1+ (支持 App Router 和 Page Router)
- React 19.2+
- TypeScript 5.9+
- Node.js 20+

**样式和UI**
- Tailwind CSS 4.1+
- Shadcn/ui (可选，Pro版本)
- 响应式设计
- 暗黑模式支持

**数据库和ORM**
- DrizzleORM 0.45+ (支持 PostgreSQL, SQLite, MySQL)
- PGlite (本地开发数据库，无需 Docker)
- Prisma Postgres (生产环境推荐)
- Drizzle Studio (数据库可视化工具)

**认证系统**
- Clerk 认证 (主推方案)
- 支持多种认证方式：
  - 邮箱密码登录
  - Magic Links (无密码登录)
  - 多因素认证 (MFA)
  - 社交登录 (Google, Facebook, Twitter, GitHub, Apple 等)
  - Passkeys (无密码登录)
  - 用户模拟 (User Impersonation)

## 项目二：Kirimase

**项目信息**
- **仓库**: https://github.com/nicoalbanese/kirimase
- **Stars**: 2,798 ⭐
- **作者**: Nico Albanese
- **定位**: Next.js CLI 脚手架工具 (类似 Rails)

### 核心概念

Kirimase 是一个命令行工具，用于快速构建 Next.js 全栈应用。它不是一个模板，而是一个代码生成器，类似于 Ruby on Rails 的 `rails scaffold` 命令。

### 主要功能

#### 1. 初始化和配置 (`kirimase init`)

**ORM 选项**
- **Drizzle ORM**
  - 支持 PostgreSQL, MySQL, SQLite
  - 自动配置 drizzle-zod (验证)
  - 自动配置 drizzle-kit (迁移管理)
  - 自动添加 npm scripts
  - 根据数据库类型生成对应配置

- **Prisma**
  - 自动配置 Prisma
  - 集成 zod-prisma (验证)

**认证选项**
- **Auth.js (NextAuth)**
  - 生成最新 Drizzle adapter
  - 生成通用登录组件
  - 包装 root layout
  - 生成认证工具函数
  - 支持 PlanetScale (无外键)

- **Clerk**
  - 完整配置文件生成
  - 包装 root layout
  - 生成认证工具函数

- **Lucia**
  - 完整配置文件生成
  - 生成 UI 和 API 路由
  - 登录/注册功能

- **Kinde**
  - 完整配置文件生成
  - 生成登录组件和路由处理器

**其他包**
- **tRPC**
  - 配置 App Router 支持
  - 客户端 tRPC 配置
  - 服务端配置 (server-invoker 模式)
  - 包装 root layout

- **Shadcn-UI**
  - 安装和配置
  - 包含 button 和 toast 组件
  - 自动添加 Toaster 到 root layout

- **Stripe**
  - 订阅支付配置
  - 完整集成设置

- **Resend**
  - 邮件服务配置

#### 2. 代码生成 (`kirimase generate`)

**Model 生成**
- 根据 SQL 类型生成 Drizzle schema
- 使用 drizzle-zod 生成 Zod schemas
- 前端和后端验证
- 生成 CRUD 查询和 mutations
- 完全类型安全
- 优化 Next.js 前端消费

**Controller 生成**
- **tRPC 路由**
  - 使用 Zod schemas 验证请求
  - 内置错误处理
  - 自动添加到 root router

- **Server Actions**
  - Next.js 原生 Server Actions
  - 类型安全

- **API Routes**
  - RESTful API 端点
  - 请求验证
  - 错误处理

**View 生成**
- 使用 Shadcn-UI 组件
- 即时 CRUD 操作
- 关系字段的 select 组件
- 日期字段的 datepicker
- **两种模式**:
  - React Hook Form + tRPC
  - 原生 React (useOptimistic + useValidatedForm)

### 非交互模式

支持完全命令行操作，无需交互式提示：

```bash
kirimase init -sf yes -pm bun --orm prisma -db pg \
  -a next-auth -ap github discord \
  -mp trpc stripe resend -cl shadcn-ui -ie yes
```

**参数说明**
- `-sf`: 使用 src 文件夹
- `-pm`: 包管理器 (npm, yarn, pnpm, bun)
- `--orm`: ORM 选择 (drizzle, prisma)
- `-db`: 数据库类型 (pg, mysql, sqlite)
- `-dbp`: 数据库提供商
- `-a`: 认证方案
- `-ap`: 认证提供商 (next-auth)
- `-mp`: 其他包
- `-cl`: 组件库
- `-ie`: 包含示例

### 工作流程

1. **初始化项目**
   ```bash
   npx create-next-app my-app
   cd my-app
   kirimase init
   ```

2. **生成资源**
   ```bash
   kirimase generate
   ```
   - 选择模型名称
   - 定义字段和类型
   - 选择关系
   - 生成 Model + Controller + View

3. **立即使用**
   - 自动生成的 CRUD 界面
   - 类型安全的 API
   - 验证和错误处理

### 设计哲学

- **Rails 风格**: 约定优于配置
- **快速开发**: 几分钟内搭建完整功能
- **最佳实践**: 内置行业标准
- **灵活性**: 支持多种技术栈组合
- **类型安全**: 端到端 TypeScript
- **可定制**: 生成的代码完全可修改

### 适用场景

- 快速原型开发
- MVP 构建
- 学习 Next.js 全栈开发
- 标准化团队开发流程
- 减少重复性代码编写

### 注意事项

- 仅支持 App Router (不支持 Pages Directory)
- 正在计划完全重写
- 当前仅修复关键 bug

### 选择建议

**选择 ts-nextjs-tailwind-starter 如果:**
- 需要干净的启动模板
- 想要完全控制项目架构
- 不需要预设的 SaaS 功能
- 重视代码质量和开发体验
- 学习 Next.js 最佳实践
- 构建简单到中型应用

**不选择此模板如果:**
- 需要快速构建 SaaS 应用
- 需要预设的认证和支付功能
- 需要数据库和 ORM 集成
- 时间紧迫，需要开箱即用的功能
